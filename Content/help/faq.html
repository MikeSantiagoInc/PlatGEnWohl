Q: Where is a "Level Settings"?

A: If you need global level settings, open in menubar [Level->Properties]. But if you need configure current section, you must open Section Properties window in menu [View -> Section Settings]

Q: Why music doesn't play while editing?
A: For play music, you need simply press to note "♪" icon on toolbar, or you can set autostart playing of music:
Open in menubar [Tools -> Application Settings], and set option [Autoplay music after opening file], and music will automatically play on each file open. If music even after this doesn't play again, please, check the data/music directory, it must be contain the SMBX's musics. If folder isn't exist, please, copy your music folder from SMBX, or download music pack from our site.

(on Microsoft Windows) If music stell don't playing, try to add new system variable:

QT_PLUGIN_PATH

and set its value equal to full path of your pge_editor's directory ( For example: C:\Games\SMBX\Pge_Editor\ )

Q: I can't find "auto-align" option!

A: This is the "snap to grid" option. For disable/enable them, please, click to sharp icon "#" on toolbar.

Q: I see ghosties instead NPCs, Why?

A: Ghost - is a dummy image sprite, It appears, if you use the broken custom image, in non-supported format, in wrong format, etc.
For fix this error, you must convert image to valid format. Supported file formats: PNG, GIF, JPG, BMP. You can add more supported image formats, if you place into the imageformats dir special plugins, what you can get from Qt. You can get them here.

Q: Do the graphics use the same format as SMBX?

A: Yes, in SMBX-compatible configs you can use graphics by SMBX64 standard:
 - GIF file format without transparence
 - Must used black/white mask for apply transparency
 - Animation frames of sprite going down by vertical way.
 - Playable character sprite have a 10x10 frame table, whole image size in 1000x1000px and fixed by position physical boxes.
 - All SMBX's custom graphics fully compatible with engine.

 

Q: Is this a new version of SMBX?

A: This project is offered as a successor of SMBX. It is not SMBX, but fully compatible with it.
See: SMBX - is a game, PGE - is a game engine. All games are created based on their own game engines.
SMBX - closed sources, PGE - Libre and Open Source.
This project can be used not only for Mario games based on SMBX, but for others too (while being entirely open source). Because it allows full flexibility and customizing (all items are defined by INI files), and NPCs will be allowed to be programmed by a script language, this allows one to create any own NPCs of their own without closing algorithms to a specific NPC-ID.
